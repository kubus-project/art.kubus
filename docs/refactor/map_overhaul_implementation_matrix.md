# Map UX, motion, responsiveness, and performance implementation matrix

Date: 2026-07-14  
Branch: `feat/map-ux-motion-overhaul`  
Baseline commit: `d76be942`

This matrix describes the current repository state after code inspection,
focused/full Flutter tests, a release web build, deterministic marker stubs,
and fresh Chromium captures. It supersedes stale implementation suggestions in
older audits; good shared infrastructure is retained.

## Workstream ownership and overlap constraints

1. SOL owns the interaction architecture, screen integration, diff review, and
   final runtime/performance acceptance.
2. Terra may implement bounded foundations or components after this matrix;
   no two agents may edit the same map screen/controller pipeline concurrently.
3. Luna remains read-only and adversarial; each concrete finding is fixed,
   rejected with evidence, or explicitly classified out of scope.
4. Cluster/sync/controller edits are sequenced after surface/filter foundations.
5. The centralized glass system, shared marker semantics, web hit testing,
   feature flags, and mobile/desktop data parity are non-regression constraints.
