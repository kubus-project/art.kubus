# Governance

## Project model

This repository is maintained as an open-source client project under Apache-2.0.
The hosted backend platform is operated separately.

## Maintainers

Maintainers review and merge pull requests, manage releases, and enforce project policies.

## Decision-making

- Small/low-risk changes: maintainer review + merge.
- Significant architectural or policy changes: proposal issue + maintainer consensus.

## Release process

- Keep changes incremental and verifiable.
- Ensure core validation passes before release.
- Document notable behavior changes in release notes.

## Conduct and enforcement

Community behavior follows `CODE_OF_CONDUCT.md`.
Security issues follow `SECURITY.md`.
